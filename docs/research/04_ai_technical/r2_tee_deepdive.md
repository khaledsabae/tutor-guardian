Arabic is a highly inflected, non-concatenative language derived from consonant-based root systems (Maddah) mapped onto semantic templates (Wazn).17 Grammatical functions, tenses, and genders are represented by vowel diacritics (Tashkeel) and attached clitics—such as coordinating conjunctions ("و", "ف"), prepositions ("بـ", "لـ"), and possessive pronouns ("ـنا", "ـكم").17
Aggressive 4-bit quantization degrades the model's high-dimensional attention mechanisms, which are needed to process these complex token features.3 This loss of precision causes attention failures, leading the model to confuse active and passive verb forms, miss negation particles, or overlook delicate theological nuances in Islamic parenting texts.3
The 8-bit quantized qwen2.5:3b preserves over 99% of the unquantized model's semantic accuracy, achieving an average Arabic score of 59.65% (base) and 64.04% (fine-tuned).3 This performance outperforms alternative architectures like Phi-4 Mini (26.60% Arabic Specific) and Gemma 2 2B (17.10% Arabic Specific) under similar resource constraints.9
## 2. Optimized Arabic RAG Subsystem
### 2.1 Tokenization & Semantic Chunking Strategy
Arabic Text Normalization Pipeline: Raw Arabic documents often contain diacritical variations, non-standard spelling, and typographical elongations (Tatweel) that complicate vector embedding lookups.18 An un-normalized text representation increases vocabulary complexity, causing semantic search models to fail when matching different typographic forms of the same word. The normalization pipeline implements the following deterministic transformations:
Diacritic Removal: Strips all vocalization diacritics, short vowels, and pronunciation markers matching the unicode range `` (including Fatha, Damma, Kasra, Tanween, Sukoon, and Shadda).20
Tatweel Pruning: Deletes any instance of the elongation symbol \u0640 (Kashida) used for line justification.22
Orthographic Standardization: Normalizes all spelling variations of Alef (أ, إ, آ, ٱ) to a bare Alef (ا).18
Suffix and Terminal Harmonization: Maps Teh Marbuta (ة) to Heh (ه) and Alef Maksura (ى) to Yeh (ي) to align varying spellings of grammatical markers.18
Whitespace Consolidation: Normalizes all sequence gaps and newlines using the regex pattern \s+ into a single space character.18
Applying this normalization pipeline reduces the unique character combination profile of Arabic corpora by approximately 50%, simplifying vocabulary representation without degrading semantic density.18 This standard vocabulary matches are then processed using WordPiece tokenization to handle the agglutinative morpho-syntax of the Arabic language.18
Chunk Size & Overlap Metrics: Arabic prose relies on coordinating conjunctions to build long, multi-clause sentences, rather than using frequent terminal punctuation.19 Traditional character-count splitters split these continuous sentences arbitrarily, severing clitics from their roots and isolating dependent clauses from their subjects.17Empirical evaluations of Arabic RAG components demonstrate that sentence-aware chunking consistently outperforms fixed-size, semantic, or recursive splitters on aggregate RAGAS metrics.23 It achieves the highest context recall and answer relevancy by maintaining the integrity of complete clauses.The system uses a target chunk size of 512 tokens with an overlap of 64 tokens, splitting text blocks on Arabic terminal punctuation markers (such as periods ., question marks ؟, exclamation points !, and semicolons ؛).20 This configuration ensures that Islamic parenting advice and behavioral instructions remain contextualized within a single chunk.

### 3.1 Local Personalization State Machine
Personalization data must be managed entirely offline to comply with the zero-telemetry requirements of the Tutor Guardian platform.25 This is done by storing child developmental parameters directly within a local SQLite database on the home network node, rather than sending behavioral metrics to external services.27

early_childhood (Ages 2-6): Maps advice to play-based learning, basic language patterns, and concrete concepts.
middle_childhood (Ages 7-12): Maps advice to logical validation, conversational reasoning, and collaborative behavior models.

By combining the child's age_group with the active educational domain and chronological progress_vector at query time, the system prompt is generated dynamically on-device, preventing the creation of analytical tracking profiles on external cloud infrastructure.27

When computational demands require migrating inference workloads from the local server to cloud setups, maintaining the zero-telemetry privacy contract requires local sanitization.29 Before any text leaves the local gateway, it passes through an offline, on-premises hybrid pre-filter designed to remove Personally Identifiable Information (PII).32

Second, to catch unstructured context leaks and non-English edge cases that rule-based systems miss, a secondary local pass is executed using the openai/privacy-filter token-classification model.36 Running locally via WebGPU/WASM, this 1.5B parameter model uses a bidirectional transformer architecture with a constrained Viterbi decoder over BIOES tags to label privacy spans (e.g., private_person, private_address, private_email, private_phone, secret) in a single forward pass.37 This hybrid approach ensures that only sanitized, anonymized text is dispatched to external endpoints.32
### 4.2 Confidential Computing TEE Serving Architecture
