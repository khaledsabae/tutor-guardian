# خطة الإنتاج والتشغيل المؤتمت لوكيل الذكاء الاصطناعي (Hermes Production Plan)

أهلاً بك يا زميلي **Hermes**. أنت مكلف بالتشغيل المؤتمت بالكامل لخط إنتاج الوسائط والدروس لتطبيق الموبايل **"المربي الذكي" (Tutor Guardian)** بالاعتماد على مساحة Google NotebookLM الخاصة بالعميل.

مهمتك هي قراءة هذه الخطة، تنفيذها بدقة، تدوين الخطوات والتحديات، وترك توثيق كامل وموثوق لكي يستطيع العميل مراجعة عملك.

---

## 🔑 المصادقة والاعتماد المسبق (Authentication & Credentials)
* **أين توجد المصادقة؟**: ملفات تعريف الارتباط وجلسة تسجيل الدخول لـ NotebookLM محفوظة بالفعل على النظام في المسار المرفق بملف المستخدم الرئيسي:
  `/home/khalednew/.notebooklm/profiles/default/storage_state.json`
* **كيف تستخدمها؟**: نظراً لأنك قد تعمل كوكيل (Agent) بـ Environment أو HOME مختلف، **يجب عليك تصدير المتغير البيئي `HOME=/home/khalednew`** أو تمريره مباشرة قبل كل استدعاء للأمر `notebooklm`.
  - **مثال**:
    ```bash
    HOME=/home/khalednew ./notebooklm_env/bin/notebooklm source list -n 94f191e6-cfbc-4655-a0d7-c8f7ad0f2287 --json
    ```
  - هذا سيوجه الأداة تلقائياً لقراءة ملف الاعتماد الموثق وتخطي خطأ `AUTH_REQUIRED`.

---

## 📋 نظرة عامة وسياق المشروع (Project Context)
* **المشروع**: تطبيق موبايل للتربية الذكية يستهدف الآباء العرب (المرحلة الحالية تركز على الجمهور المصري).
* **المساحة على NotebookLM**: `94f191e6-cfbc-4655-a0d7-c8f7ad0f2287`.
* **الهدف**: توليد 4 فئات من الوسائط لكل درس من الدروس المرفوعة:
  1. **ملخص دقيقة واحدة (Quick Summary)**: مؤصل علمياً بالنصوص.
  2. **فلاش كاردز تفاعلية (Flashcards)**: مواقف وحلول في 3 خطوات.
  3. **سلايدات الجلسات العائلية (Family Slides)**: شرائح بها أنشطة وأسئلة تفاعلية.
  4. **بودكاست صوتي (🎧)**: حلقة حوارية بين مقدم ومقدمة بالعامية المصرية الدافئة (`ar_eg`) يسهل الاستماع لها.
* **التأصيل الصارم (Strict Grounding)**: يُمنع تماماً اختراع معلومات عامة أو نصائح من الإنترنت؛ يجب الاعتماد الكلي والكامل على محتوى المرجع المرفوع حصرياً.

## 🛠️ المهام التقنية المطلوبة منك (Technical Execution Steps)

لديك 5 مراحل يجب تنفيذها حلقة تكرارية (Loop) على كافة الدروس المرفوعة في المساحة:

### المرحلة 1: جرد قائمة الدروس ومصادرها (Source Inventory)
1. قم بتشغيل الأمر التالي (مع تمرير متغير HOME) للحصول على قائمة المصادر المرفوعة بالكامل ومعرفاتها (Source IDs):
   ```bash
   HOME=/home/khalednew ./notebooklm_env/bin/notebooklm source list -n 94f191e6-cfbc-4655-a0d7-c8f7ad0f2287 --json
   ```
2. قم بتحديد الدروس التي تبدأ بصيغة المنهج الجديد `age_*` (مثلاً: `age_4_6 - lesson_...` أو `age_16_18 - lesson_...`).

### المرحلة 2: توليد الأصول النصية المؤصلة (Text Assets Generation)
لكل درس مستهدف، قم بإرسال استعلامات فردية ومباشرة باستخدام معرف المصدر (`-s SOURCE_ID`) لحفظ المخرجات في مجلد `docs/` بصيغة Markdown تحت اسم `lesson_<age_group>_<domain>_<name>_assets.md`.

* **أمر توليد الملخص (Summary Prompt)**:
  ```text
  قم بتلخيص الدرس المحدد بأسلوب "القراءة السريعة في دقيقة واحدة". للدرس، استخرج النقاط التالية بدقة بالغة وبأسلوب نقاط (Bullet points) جذاب:
  1. الخلاصة الكبرى: الفكرة الأساسية للدرس في جملة واحدة مكثفة.
  2. المحفز السلوكي/العلامة: ما هي العلامة أو السلوك الذي يجب أن ينتبه إليه الأهل؟
  3. خطوات اليوم: 3 إجراءات عملية يمكن للأب أو الأم البدء بها حالاً.
  4. شواهد التأصيل: عبارة أو اقتباس مباشر من الدرس يؤكد هذه الاستراتيجية.
  شروط التوليد الصارمة: اعتمد كلياً وحصرياً على الدرس المحدد ولا تضف أي فكرة خارجية أو نصيحة عامة.
  ```

  **أمر تشغيل توليد الملخص**:
  ```bash
  HOME=/home/khalednew ./notebooklm_env/bin/notebooklm ask -n 94f191e6-cfbc-4655-a0d7-c8f7ad0f2287 -s SOURCE_ID --new -y "نص أمر التلخيص هنا"
  ```

* **أمر توليد الفلاش كاردز (Flashcards Prompt)**:
  ```text
  بناءً على الدرس المحدد حصرياً، قم بتوليد مجموعة من الفلاش كاردز (Flashcards) التفاعلية للأهل.
  كل بطاقة يجب أن تتبع الهيكل التالي بدقة وتكتب بلغة عربية مبسطة وواضحة جداً:
  - المجال والفئة العمرية
  - الوجه الأول (الموقف التربوي/السؤال): موقف واقعي محفز للتفكير أو سؤال مباشر يواجهه الأهل في الحياة اليومية.
  - الوجه الثاني (الإجراء السريع - Action Plan): 3 خطوات عملية مكتوبة باختصار شديد لتطبيق الحل فوراً.
  شروط التوليد الصارمة: اعتمد كلياً وحصرياً على المرجع المحدد ولا تضف أي نصائح عامة.
  صغ المخرجات في جدول Markdown نظيف ومقسم كالتالي:
  | الفئة والمجال | الوجه الأول (السؤال/الموقف) | الوجه الثاني (الحل في 3 خطوات) |
  ```

  **أمر تشغيل توليد الفلاش كاردز**:
  ```bash
  HOME=/home/khalednew ./notebooklm_env/bin/notebooklm ask -n 94f191e6-cfbc-4655-a0d7-c8f7ad0f2287 -s SOURCE_ID --new -y "نص أمر الفلاش كاردز هنا"
  ```

* **أمر توليد الشرائح العائلية (Slides Prompt)**:
  ```text
  قم بتحويل الدرس المحدد إلى "حقيبة عرض عائلي تفاعلي" مقسمة كشرائح عروض تقديمية (Slides). 
  لكل شريحة، حدد بوضوح:
  - عنوان الشريحة: عنوان مشوق وجاذب للأبناء.
  - المحتوى البصري المقترح: وصف لما يجب رسمه أو إظهاره في الشريحة.
  - الرسالة الأساسية: الفكرة التي يشرحها الأب أو الأم بأسلوب مبسط للأولاد.
  - سؤال للنقاش العائلي: سؤال مفتوح يوجه للأبناء لفتح باب الحوار والاستماع إليهم.
  - نشاط عملي جماعي: لعبة أو نشاط حركي مدته 5 دقائق يرسخ قيمة الدرس في الجلسة.
  شروط التوليد الصارمة: اعتمد كلياً وحصرياً على المرجع المحدد ولا تضف أي نصائح عامة.
  ```

  **أمر تشغيل توليد الشرائح**:
  ```bash
  HOME=/home/khalednew ./notebooklm_env/bin/notebooklm ask -n 94f191e6-cfbc-4655-a0d7-c8f7ad0f2287 -s SOURCE_ID --new -y "نص أمر الشرائح هنا"
  ```

### المرحلة 3: توليد وتنزيل البودكاست بالعامية المصرية (Audio Overview Generation)
1. أرسل أمر توليد البودكاست مع تفعيل خيار العامية المصرية والانتظار:
   ```bash
   HOME=/home/khalednew ./notebooklm_env/bin/notebooklm generate audio -n 94f191e6-cfbc-4655-a0d7-c8f7ad0f2287 -s SOURCE_ID --prompt "تصرفوا كمقدمي بودكاست تربوي محترفين وودودين (رجل وامرأة)، يتناقشان بالعامية المصرية الراقية والسهلة. شروط الحوار الصارمة: اعتمدوا بنسبة 100% وبشكل استحواذي على محتوى الدرس المحدد. ممنوع تقديم نصائح عامة. قسّما النقاش إلى نصائح عملية قابلة للتطبيق الفوري اليوم في المنزل مستخلصة مباشرة من المرجع." --language ar_eg --wait --json
   ```
2. قم بتنزيل الملف الصوتي وحفظه في مجلد `docs/` بصيغة MP3:
   ```bash
   HOME=/home/khalednew ./notebooklm_env/bin/notebooklm download audio -n 94f191e6-cfbc-4655-a0d7-c8f7ad0f2287 docs/lesson_<age>_<domain>_podcast.mp3 --latest --force
   ```

### المرحلة 4: الهيكلة البرمجية للوسائط (Parsing Markdown to JSON)
قم بتشغيل سكربت التحليل التلقائي على المخرجات النصية لاستخراج جداول الفلاش كاردز ونقاط الشرائح وحفظها كملفات JSON جاهزة لتغذية قاعدة بيانات التطبيق:
```bash
./scripts/parse_generated_assets.py
```
*(ملاحظة: يمكنك توسيع السكربت أو تكرار المنطق ليشمل كافة الملفات المصدرة).*

## 📝 سجل الإنتاج والتوثيق المطلوب منك (Production Registry)

يجب عليك إنشاء وتحديث ملف مركزي باسم **`docs/production_registry.md`** ليكون سجلاً لتقدم العمل.
لكل درس تقوم بمعالجته، يجب أن تسجل المدخلات التالية بدقة:

```markdown
### [اسم الدرس والرمز المعرف]
- **معرف المصدر (Source ID)**: `...`
- **حالة التوليد النصي**: [ناجح / فشل]
- **حالة البودكاست الصوتي**: [ناجح / فشل]
- **عنوان حلقة البودكاست المولّدة تلقائياً**: "..." (مثال: بلاش برافو وامدح مجهود طفلك صح)
- **حجم الملف الصوتي المُنزل**: `... MB`
- **حالة التحليل إلى JSON**: [ناجح / فشل]
- **الملاحظات والتحديات التي واجهتها**: (مثال: البطء في توليد البودكاست، أو مشاكل في استجابة الـ API أو هيكلة الجداول المwrapped).
```

## 🚦 شروط وإرشادات الاسترداد وإدارة الأخطاء (Error Handling)
1. **التعامل مع البطء (Rate Limits & Timeout)**: توليد البودكاست الصوتي قد يستغرق من 7 إلى 12 دقيقة للدرس الواحد. لا تقم بإلغاء العملية؛ انتظر حتى يعود كود الحالة بـ `completed` أو قم بالتحقق من آخر صوت تم توليده وتنزيله بـ `--latest`.
2. **فشل التحليل النصي**: إذا تغير تنسيق مخرجات الفلاش كاردز أو الشرائح ولم يتمكن سكربت الـ Parser من التقاطها، قم فوراً بتسجيل ذلك في حقل "التحديات والملاحظات" في السجل المركزي، وقم بتعديل تعبيرات الـ Regex في السكربت لتطابق الهيكل الجديد.
3. **تكرار الملفات**: إذا حدث خطأ وأدى إلى تكرار الملفات في مساحة العمل، استخدم السكربت المخصص `scripts/clean_notebooklm_duplicates.py` لتنظيف المساحة فوراً والإبقاء على النسخ الأحدث فقط.
